The Monty Hall Problem

I want to devote the first post of this blog to the infamous Monty Hall problem in probability. I’m sure most of the readers have come across with this puzzle before. Some of you probably thinks this is cliché when you see the title. However, it was not until very recently that I only discovered some of the subtleties in this problem. If you think the answer is simply “always switch because the probability is 2/3,” this post will be worth your time reading.

So the puzzle goes like

Suppose you’re on a game show, and you’re given the choice of three doors: Behind one door is a car; behind the others, goats. You pick a door, say No. 1, and the host, who knows what’s behind the doors, opens another door, say No. 3, which has a goat. He then says to you, “Do you want to pick door No. 2?” Is it to your advantage to switch your choice?

Source: Monty Hall Wikipedia

The important assumption here is that the host will always open a door with a goat that you did not pick.

Most readers probably already know the standard answer: if you stay, your chance of winning is \(1/3\), but if you switch, your chance of winning is \(2/3\). Here is the intuitive argument: the event that I win the car if I switch is the same as the event that the car is behind door 2 or 3. Hence, the chance of winning if you switch is 2/3.

However, this argument actually only proves the marginal probability of winning. It tells you that if you commit to a switching strategy beforehand, you will win the car \(2/3\) of the time.

What it doesn’t tell you is the conditional probability. Once you are actually on the show and the host has specifically chosen to open Door 3, is your chance of winning by switching to Door 2 still exactly \(2/3\)?

As I will show you below, this is in fact just not true in general.

Before going into the theory. Let’s run a some simulations to see if it is advantageous to switch, and you will see what I mean.

Simulation

How do we simulate this game? Each time we will uniformly randomly pick a number in \(\{1, 2, 3\}\) to represent the door with the car behind it.

import numpy as np

rng = np.random.default_rng() # Random number generator
N = 10000 # Number of simulations

door_with_car = rng.integers(1, 4, size=N)

Then we need to figure out which door will get revealed by the host. We will just assume that we choose door 1 in every simuation. Then how does the host act? If the car is behind door 2, then the host will open door 3. If the car is behind door 3, the host will open door 2. However, what if the car is behind door 1? In this case we don’t know if the host will open door 2 or door 3. We do not know the bias of the host among door 2 and 3.

So it turns out our knowledge of the host is not enough for us to simulate their behavior. What we can do is parametrize the distribution of host action in the case where the car is behind door 1. We let \[\rho = \mathbb P(\text{host opens door 2} \mid \text{car is behind door 1}).\]

def host_action(door_with_car, rho=0.3):
    # Host opens a door that has a goat and is not the player's choice
    host_opens = np.zeros_like(door_with_car)
    # if car is behind door 1, host opens door 2 with probability rho and door 3 with probability 1-rho
    host_opens[door_with_car == 1] = rng.choice([2, 3], size=np.sum(door_with_car == 1), p=[rho, 1-rho])
    # if car is behind door 2, host opens door 3
    host_opens[door_with_car == 2] = 3
    # if car is behind door 3, host opens door 2
    host_opens[door_with_car == 3] = 2
    return host_opens

host_opens = host_action(door_with_car)
stay_wins = np.sum(door_with_car == 1)
print(f"Probability of winning by switching: {1 - stay_wins / N:.2f}")
Probability of winning by switching: 0.67

Ok, so when \(\rho = 0.7\), the chance of winning the car if switch is \(2/3\) which seems to be consistent with the result we all know, and independent of the value of \(\rho\). However, this is over all the experiments, not those where the host opens door 3. If instead we only look at cases where the host opens door 3, we will see that

stay_wins_if_host_opens_3 = np.sum((door_with_car == 1) & (host_opens == 3))
host_opens_3 = np.sum(host_opens == 3)
print(f"Probability of winning by switching given host opens door 3: {1 - stay_wins_if_host_opens_3 / host_opens_3:.2f}")
Probability of winning by switching given host opens door 3: 0.60

Ok this is interesting. The chance of winning if switch given that the host opens \(3\) is actually lower than \(2/3\), which seems to depend on the value of \(\rho\). Let’s plot it against \(\rho\).

rho_dense = np.linspace(0, 1, 500)
p_switch_wins_given_host_opens_3 = np.zeros_like(rho_dense)
p_switch_wins = np.zeros_like(rho_dense)
for i, rho in enumerate(rho_dense):
    host_opens = host_action(door_with_car, rho)
    stay_wins_if_host_opens_3 = np.sum((door_with_car == 1) & (host_opens == 3))
    stay_wins = np.sum(door_with_car == 1)
    host_opens_3 = np.sum(host_opens == 3)
    p_switch_wins_given_host_opens_3[i] = 1 - stay_wins_if_host_opens_3 / host_opens_3 if host_opens_3 > 0 else 0
    p_switch_wins[i] = 1 - stay_wins / N

import plotly.graph_objects as go

fig = go.Figure()

# Trace 1: Conditional probability
fig.add_trace(go.Scatter(
    x=rho_dense, 
    y=p_switch_wins_given_host_opens_3, 
    mode='lines', 
    name="P(switch wins | host opens door 3)"
))

# Trace 2: Marginal probability
fig.add_trace(go.Scatter(
    x=rho_dense, 
    y=p_switch_wins, 
    mode='lines', 
    name="P(switch wins)"
))

# Layout updates with unicode for rho
fig.update_layout(
    title="Probability of Winning by Switching vs. Host's Bias",
    xaxis_title="Host's bias ρ (probability of opening door 2 when car is behind door 1)",
    yaxis_title="Probability of Winning by Switching",
    legend_title="Legend"
)

fig.show()

So, the probability of winning if switch actually positively correlates with the host’s bias of opening door 2, given that he host opens door 3. This is in fact intuitively clear: in the extreme case, when \(\rho = 1\), if the car is behind door 1, the host would never open door 3; so we know the car is not behind door 1 given that the host opened door 3. Hence we are guaranteed to win the car if we switch. This argument also generalizes to cases when \(\rho\) is close to \(1\) if you think about it.

Theory

Now let’s make some theoretical analysis. The setup is that \(X \sim \text{uniform}(\{1, 2, 3\})\) is the door with car, and \(Y\) is the door that the host opens. Then \(Y = 2\) if \(X = 3\), \(Y = 3\) if \(X = 2\), and \(\mathbb P(Y = 2 \mid X=1) = 1- \mathbb P (Y=3 \mid X=1) = \rho\). The probability of winning if you switch is: \[1 - \mathbb P(X = 1 \mid Y = 3) \]

Using the Bayes rule, we can expand the conditional probability: \[1- \mathbb P(Y = 3 \mid X = 1)\frac{\mathbb P(X = 1)}{\mathbb P(Y = 3)}\]

Plugging in our values, this gives us: \[1 - (1 - \rho)\frac{1/3}{1/3 + (1-\rho)/3} = \frac{1}{2 - \rho}\]

This isn’t much effort actually! However, if one is not careful it is very easy to come up with an incorrect solution; most of the times because they fail to realize that the host’s action is not fully specified. I will not include one such example here for the sake of brevity.

Summary

People are often confused by probability puzzles because the problems themselves are not fully specified. Or to put it in other words, because you do not have perfect information of the dynamics of the environment. In this example, that is not to say you do not know which door is with the car; it is that you do not have information of the distribution of the host’s action. And often the easiest way to find out if you know all the dynamics/distribution is to try to simulate the environment. By simulation you automatically figures out what you don’t know yet (because you have to specify the distribution when generating the data). Therefore, running simulations, or just thinking about how one could simulate the puzzle, is often very useful.
